El punto de partida

¿Qué problema resuelve?

Sin el sistema
  • Las solicitudes se acumulan y nadie las lee a tiempo
  • Cada persona clasifica con un criterio distinto
  • Las preguntas repetidas ocupan a los técnicos
  • Las emergencias no se distinguen del resto
Con el Help Desk Inteligente
  • La IA lee y clasifica cada ticket en segundos
  • Criterio uniforme: siempre las mismas reglas
  • Las preguntas frecuentes se responden solas
  • Lo urgente se escala a Trello al instante y todo se notifica por correo
Los tres modelos de servicio

IaaS · PaaS · SaaS en un solo proyecto

IaaS

Infraestructura

Servidores, red y almacenamiento que ponen los proveedores

  • Cómputo serverless sobre AWS
  • Clúster MongoDB en São Paulo
  • Replicado en 3 nodos: si uno cae, sigue vivo
PaaS

Plataforma

Desplegar código sin administrar servidores

  • Vercel ejecuta frontend + API
  • Cada push a GitHub = deploy automático
  • MongoDB Atlas como base de datos gestionada
SaaS

Software listo

Servicios terminados que se consumen por API

  • Gemini: el agente de IA
  • Trello: tablero de emergencias
  • Brevo: correos · GitHub: colaboración

Datos y protección

Persistencia y seguridad

Datos

🗄️ MongoDB Atlas

Colecciones tickets y logs (auditoría del agente). Replicación real en 3 nodos: alta disponibilidad sin configurar nada.

Secretos

🔑 7 llaves protegidas

Mongo, Gemini, Trello, Brevo y la clave admin viven como variables de entorno en Vercel. El repositorio público no expone nada.

Acceso

🔒 Autenticación simple

Cerrar o eliminar tickets exige una clave validada en el servidor. Clave incorrecta = acción denegada.

Defensa

🛡️ Validaciones y anti-XSS

El servidor valida correos, longitudes e IDs. Todo texto de usuario se muestra como texto plano: nadie inyecta código.
